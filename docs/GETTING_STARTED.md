# Начало работы с Scenaria

Краткое руководство для тех, кто впервые открывает приложение.

## Интерфейс

| Область | Назначение |
|---------|------------|
| **Вкладка «Старт»** | Быстрый старт, недавние файлы, ссылка на примеры |
| **Дерево слева** | Проект — папка с `.feature` файлами |
| **Панель действий** | Кнопки: браузер, запись, сохранение, запуск теста |
| **Chip с именем файла** | Какой сценарий сейчас открыт (скрыт на «Старте») |
| **URL** | Стартовый адрес для записи и новых сценариев |
| **Редактор** | Текст Gherkin; **Ctrl+Space** — подсказки шагов |
| **Справка** | **F1** — каталог шагов, таблицы примеров, TestClient |

Меню сгруппировано по workflow: **Проект**, **Сценарий**, **Запись и тест**, **Плагины**, **Вид**, **Справка**.

## Первый сценарий за 5 минут

### Вариант А: примеры из коробки

1. **Старт** → **Открыть примеры сценариев**
2. Выберите `01-pervaya-proverka.feature`
3. **Запустить тест** (Ctrl+Enter) — откроется браузер, сценарий выполнится

Для знакомства с TestClient откройте `05-testclient-kontekst.feature` (см. [examples/README.md](../examples/README.md)).

### Вариант Б: запись своих действий

1. **Старт** → введите URL сайта → **Быстрый старт** (или **Сценарий → Новый**)
2. **Открыть браузер** (Ctrl+B) → **Начать запись** (Ctrl+R)
3. Кликайте и вводите текст на сайте
4. **Остановить запись** — шаги появятся в редакторе
5. **Сохранить** (Ctrl+S) → **Запустить тест**

## Рабочий процесс (подробно)

1. **Проект → Открыть проект…** — папка с `.feature`
2. **Сценарий → Новый** или открыть файл из дерева
3. **Открыть браузер** → при необходимости **Начать запись**
4. **Остановить запись** — шаги в редакторе
5. **Сохранить** (Ctrl+S)
6. **Запустить тест** (Ctrl+Enter) — неприменённые правки применяются автоматически

Баннер **«Применить»** означает, что текст в редакторе ещё не разобран в модель. При запуске теста применение происходит само; вручную: **Ctrl+Shift+S**.

## TestClient (сохранённая сессия браузера)

Если сценарий требует входа на сайт, сохраните cookies и localStorage один раз и подключайте профиль в Gherkin:

```gherkin
Контекст:
	Дано я подключаю TestClient "ИмяПрофиля"

Сценарий: Работа под пользователем
	Допустим открыт "https://example.com"
	…
```

**Как настроить:**

1. **Запись и тест → TestClient…** — откройте браузер, войдите на сайт, **Сохранить текущий TestClient…**
2. Профили хранятся в `.scenaria/test_clients/<имя>.json` внутри папки проекта
3. Имя в кавычках в Gherkin должно совпадать с именем файла
4. Без блока `Контекст:` — чистый браузер при каждом прогоне

Справка с примером вставки: **F1** → категория **«Сессия и TestClient»**.

## Шаги Gherkin

Сценарии пишутся **по-русски**. Ключевые слова: `Допустим`, `Когда`, `Тогда`, `И`, `Но`.

| Категория | Примеры |
|-----------|---------|
| Навигация | `открыт "url"`, `обновляю страницу`, `возвращаюсь назад`, `скроллю к` |
| Формы и ввод | `нажимаю`, `ввожу`, `выбираю`, `отмечаю`, `загружаю файл` |
| Генераторы | `ввожу случайный телефон`, `ввожу "{{first_name}}"` |
| Ожидание | `жду 2 сек`, `жду появления`, `жду исчезновения` |
| Проверки | `вижу`, `не вижу`, `проверяю текст`, `проверяю url` |
| Условия | `Если вижу "…"` + вложенные шаги с отступом |
| Циклы | `Повторяю 3 раза`, `Пока вижу "…"`, `Для каждого "…" как "var"` |
| Сессия | `закрываю браузер`; `Контекст:` + TestClient |

Полный каталог: **Справка → Справка…** (F1) или **Ctrl+Space** в редакторе.

### Вложенные блоки (условия и циклы)

Вложенные шаги пишутся **с отступом** (таб или два пробела) и с ключевым словом `И`:

```gherkin
Если вижу ".cookie-banner"
	И нажимаю "button.accept"
Повторяю 3 раза
	И нажимаю "button.add"
```
